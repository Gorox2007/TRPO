# 5. Этап 3: Анкеты, ранжирование и кэширование

Этап 3 реализует рабочий сценарий dating-бота поверх архитектуры `Bot -> Core -> PostgreSQL`.

## 5.1 CRUD анкет

Реализовано:
- `create/update`: команда `/profile_set birth_date=2000-01-31 gender=female city=Москва bio="Люблю кофе"`;
- `read`: команда `/profile`;
- `delete`: команда `/profile_delete`;
- фото: отправка изображения в чат или тестовая команда `/photo_add file_id=... unique_id=...`;
- автоматический пересчет `profile_completeness` после обновления анкеты или фото.

Полнота анкеты считается по пяти признакам:
- дата рождения;
- пол;
- город;
- описание `bio`;
- хотя бы одно активное фото.

## 5.2 Алгоритм ранжирования

Алгоритм находится в `stage2_bot/core.py`.

Используются факторы из двух уровней задания:
- уровень 1, первичный рейтинг: возрастной диапазон, пол, город, полнота анкеты, количество фото;
- уровень 2, поведенческий рейтинг: соотношение лайков и пропусков, количество мэтчей.

Формула:

```text
total_score = 0.70 * primary_score + 0.30 * behavioral_score
```

`/next` показывает кандидата с итоговым score и детализацией `primary`, `behavior`.

## 5.3 Кэширование анкет

Для учебного MVP используется `RecommendationCache` в `stage2_bot/cache.py`.

Кэш хранит заранее отсортированную очередь кандидатов на пользователя:
- при первом `/next` строится рейтинг и кэшируется пачка кандидатов;
- следующие `/next` берут кандидатов из кэша без повторного полного ранжирования;
- TTL по умолчанию: `300` секунд;
- размер пачки по умолчанию: `10` кандидатов;
- после изменения анкеты, предпочтений или реакции пользователя кэш пользователя инвалидируется.

Параметры запуска:

```bash
python3 -m stage2_bot.run --cache-ttl 300 --recommendation-batch-size 10
```

Такой интерфейс можно заменить на Redis без изменения Telegram-слоя.

## 5.4 Интеграция с ботом

Команды `/next`, `/like`, `/skip` больше не являются заглушками:
- `/next` получает следующего кандидата из кэша или строит новый рейтинг;
- `/like` сохраняет действие в `user_actions`;
- если второй пользователь уже лайкнул первого, создается запись в `matches`;
- `/skip` сохраняет пропуск и исключает кандидата из дальнейшей выдачи текущему пользователю.

## 5.5 Обновления БД

`db/schema.sql` сделан идемпотентным:
- добавлены поля Telegram-регистрации `telegram_id`, `first_name`, `last_name`;
- профильные поля могут быть пустыми до заполнения анкеты;
- добавлены индексы для поиска пользователей, действий и мэтчей;
- добавлен уникальный индекс на пару пользователей в `matches`.
